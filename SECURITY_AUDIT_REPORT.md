## Issues Found & Fixed

### 🔴 CRITICAL — Fixed

#### 1. Missing Firestore Rules for Top-Level Collections
**Files:** `firestore.rules`

The following collections had **NO Firestore security rules** and were wide open to any authenticated user:
- `symptom_logs` — Health data (LGPD-sensitive)
- `items` — Medication data
- `subscriptions` — Payment data
- `notification_preferences` — Notification settings
- `user_profiles`, `health_insights`, `health_history`, `medical_exams`
- `compartilhamentos_doc`, `eventos_saude`, `documentos_saude`
- `profiles`, `schedules` (top-level), `stock` (top-level)

**Fix:** Added owner-based access control rules for all missing collections.

### 🟠 HIGH — Fixed

#### 2. `sendDoseNotification` Missing Authorization Check
**File:** `functions/src/index.ts` (line ~713)

Any authenticated user could send push notifications to ANY other user.

**Fix:** Added `if (context.auth.uid !== userId)` check with `permission-denied` error.

#### 3. `generateMonthlyReport` Used Wrong Field Names
**File:** `functions/src/index.ts` (lines ~1437-1488)

Used `user_id` and `due_at` (snake_case) but actual fields are `userId` and `dueAt` (camelCase). Also used `collectionGroup` unnecessarily.

**Fix:** Changed to `.collection('dose_instances')` with `.where('userId', '==', uid)` and `.where('dueAt', '>=', startDate.toISOString())`.

### 🟡 MEDIUM — Fixed

#### 4. `useAuditLog` Wrong Path
**File:** `src/hooks/useAuditLog.ts`

Wrote to `audit_logs` but Firestore rules only allow `auditLogs`.

**Fix:** Changed path from `audit_logs` to `auditLogs`.

#### 5. `DataExport.tsx` Called Non-Existent Cloud Function
**File:** `src/pages/DataExport.tsx`

Called `exportUserData` Cloud Function which doesn't exist.

**Fix:** Replaced with client-side data fetching using `fetchDocument` and `fetchCollection`.

#### 6. `scheduleDoseNotifications` Missing `notificationSent` Filter
**File:** `functions/src/index.ts` (line ~740)

Query fetched all scheduled doses including already-notified ones.

**Fix:** Added `.where('notificationSent', '==', false)` to the query.

#### 7. `claraWeeklySummary` Used Legacy Collection
**File:** `functions/src/index.ts` (line ~1245)

Queried `users/{uid}/doses` with `scheduledTime` field (both legacy).

**Fix:** Changed to `dose_instances` collection with `dueAt` field.

## Remaining Recommendations (Non-Critical)

### 🟡 MEDIUM

1. **Add rate limits to `voiceToText`, `checkInteractions`, `claraWeeklySummary`, `claraConsultationPrep`**
   These endpoints call Gemini API but have no rate limiting.

2. **Complete `onUserDelete` cleanup**
   The auth trigger deletes some subcollections but misses many top-level collections like `symptom_logs`, `dose_instances`, etc.

3. **Remove or validate `apiToken` in `sendWhatsappReminder`**
   Accepting API tokens from client-side is a security risk.

### 🟢 LOW

4. **Document data retention policy**
   Add TTL policies for old `dose_instances`, `symptom_logs`, `auditLogs`.

5. **Add `pharmacyPrices` disclaimer**
   Prices are randomly generated — should be marked as simulated.

6. **Validate Google Calendar OAuth scope**
   Check that the access token has calendar scope before making API calls.
